# セキュリティと安全な運用マニュアル

このファイルは、Webサイトを安全に運営・公開し続けるために、「初心者が絶対に気をつけるべきポイント」をまとめたものです。

## 1. APIキーの管理（最重要）
将来的に、地図（Google Maps）やAI（ChatGPTなど）の機能を組み込む場合、**APIキー**というパスワードのようなものが必要になります。

### 🚨 絶対にやってはいけないこと
HTMLやJavaScriptのファイルの中に、APIキーを直接書き込むこと。
```javascript
// × 危険！世界中の人がこのキーを盗んで、あなたのクレジットカードで課金できてしまいます。
const apiKey = "AIzaSyD..."; 
```

### ✅ 正しいやり方
1. **環境変数を使う**: 開発ツール（ViteやNext.jsなど）を使う場合は、`.env`という隠しファイルにキーを保存します。
2. **制限をかける**: Google Cloud Consoleなどの管理画面で、**「このWebサイト（あなたのドメイン）からしか使えない」**という制限（HTTPリファラー制限）を必ず設定してください。

## 2. リンクの安全性
外部サイト（Instagram, Note, Googleフォームなど）へのリンクには、セキュリティ対策が必要です。

### ✅ 安全なリンクの書き方
`target="_blank"`（新しいタブで開く）を使うときは、必ず `rel="noopener noreferrer"` をセットで書きます。
```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">リンク</a>
```
- **noopener**: 新しいタブのページが、元のページ（あなたのサイト）を操作できないようにします。
- **noreferrer**: どのページから来たかという情報を相手に送らないようにします（プライバシー保護）。

※ 現在のあなたのサイトのリンクは全てこの対策済みです。

## 3. コードの公開・バックアップ（Git/GitHub）
もしGitHubなどにコードをアップロードする場合は、以下のファイルが含まれないように注意してください。

- `.DS_Store` (Macのシステムファイル：フォルダ構成などがバレる可能性があります)
- `.env` (APIキーなどの機密情報)
- 個人情報が書かれたメモ書きなど

これらを防ぐために、`.gitignore` というファイルを作って、除外設定をすることをお勧めします。

## 4. サーバー選び（HTTPS）
Webサイトを公開する際は、**HTTPS（URLの左に鍵マークがつく）**に対応したサーバーを選んでください。
- **おすすめ**: Vercel, Netlify, GitHub Pages
- 通信が暗号化され、見る人にとっても安全です。
